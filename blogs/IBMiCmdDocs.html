First time run

When you first run Notepad++ with IBMiCmd, you will be prompted that a new config has been created for the plugin.

The config will be created where all config is stored for Notepad++, this will usually be in your %appdata%. For example: C:\Users\youruser\AppData\Roaming\Notepad++\plugins\ConfigIBMiCmd.cfg. You can open this file up with any text editor.

Open the config and specifiy which system to use, as well as a username and password for that system. Save, close the file and restart Notepad++.

Command Entry

To open the Command Entry window:

  1. Select Plugins
  2. Select IBMiCmd
  3. Select IBM i Command Entry
There are two sections to this window. The textbox where you can enter commands and the command log section. Each set of commands you submit will have it's own job. To have multiple commands run on the same job, seperate each command a by line. It's important to note that each job will have a starting current directory of /QSYS.lib and you can use CD [dir] to change it. You should also submit a CHGLIBL command to specify a library list if required.

For example, if you wanted to compile code in your home directory you might submit 

CD '/home/USER'
CRTBNDRPG PGM(MYLIB/mypgm) SRCSTMF('mypgm.rpgle') OPTION(*EVENTF)

When entering commands, use Enter to go to the next line. To submit entered commands, use Ctrl+Enter.

Error Listing

To open the Error Listing window:

  1. Select Plugins
  2. Select IBMiCmd
  3. Select IBM i Error Listing

The Error Listing window is used to display error from a program compile (RPG, C/C++, COBOL). To do this, we have to double click on the "Double click to change object" item, which will load a popup.

In this window, we must specify the name of the program/module we tried to create and what library we tried to create it in; then press Select.

That should cause some loading to occur and then for the Error Listing to be populated. You will be notified if there are no errors. You are able to double click on the errors to be taken to the line of the error. (Note: this functionality is buggy with SQL programs.)

You can double click the "Double click to change object" to load different errors for another program.

Command Bindings

To open the Command Entry window:

  1. Select Plugins
  2. Select IBMiCmd
  3. Select IBM i Command Bindings

Command bindings are used to automate command entry and error listing. For example, a binding can do multiple takes in one binding; like change the directory, then compile a program and then load the errors for that program automatically. You will automatically have two bindings which you can use as an example.

To add a binding, press "Add bind". To edit a binding, select the binding from the list and then press "Edit bind". To delete a binding, select the bind and then use the delete key.

Double clicking on a binding will run the commands on the connected system and print the results on the Command Entry window.

Binding provides a special command, called ERRORS. This allows you to automatically display the errors of a program/module, instead of you having to select the program/module from the Error Listing screen. ERRORS has two parameters: the first is the library and the second is the program/module name. For example, in this script it will change the current directory, create a program and then load the errors for the program: 

CD '/home/LALLAN'
CRTBNDRPG PGM(#LALLAN/BUILD) SRCSTMF('RelicPackageManager/QSOURCE/BUILD.SQLRPGLE') OPTION(*EVENTF) REPLACE(*YES)
ERRORS #LALLAN BUILD
